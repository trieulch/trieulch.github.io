E-Commerce Analytics | End-to-End SQL Business Analysis

Overview

This project performs a comprehensive end-to-end analysis of an e-commerce business using MySQL, Python replicating real-world analytical workflows used by Data Analysts in technology companies.

The dataset contains realistic transactional, behavioral, and marketing data, enabling deep analysis across:

  • Traffic acquisition
  • Website performance
  • Conversion funnel optimization
  • Marketing channel performance
  • Product performance
  • User retention
  • Business growth and seasonality

    Information:

  • Targeted Conversion Rate: 4%
  • Cost Per Conversion: 2
  • Sales funnel: /home -> /products-> /(product-name) -> /cart -> /shipping -> /billing /thanks
  • 4-product launch period: mrfuzzy, lovebear, birthday panda, mini-bear

The goal is to answer stakeholder questions and generate insights to support marketing, product, and executive decision-making.

Dataset Structure

  • website_sessions — traffic source, device, and session information, 500k+ rows
  • website_pageviews — user navigation behavior, 1m3+ rows
  • orders — transaction records
  • order_items — product-level purchase details
  • order_items_refund — product-level return details

4. Business Patterns & Seasonality

Overview Business Performance

Analyzing sessions by week-year segments, we observe consistent year-over-year growth. Notable spikes appear during Black Friday and Cyber Monday(Week 47). Overall, annual sessions increased by 93.75%, while annual orders grew by 157.19%.
Then I do a basic correlation analysis between sessions and conversion rate. R² = 0.4414 indicates a moderate correlation. However, this is only a preliminary indication. We did not further investigate this relationship, as it may be influenced by multiple factors such as seasonal trends, customer type (new vs. returning), promotions, and traffic channels.
